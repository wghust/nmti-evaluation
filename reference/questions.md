# NMTI 答题集设计（情境选择版）

本答题集用于识别用户的 NMTI 牛马人格类型。

本版从“同意度量表”改为“职场情境四选一”。用户不再重复点击同一套选项，而是在具体职场场景中选择最像自己的反应。

设计目标：

* 更有代入感，让用户读题时产生“这不就是我吗”的感受
* 覆盖员工与老板、同事、跨部门之间的沟通冲突
* 保留 NMTI 四维二分结构，仍能稳定映射到 16 种人格
* 适合小程序轻娱乐测评和分享传播

---

# NMTI 维度结构

| 维度 | 方向 1 | 方向 2 | 识别重点 |
| --- | --- | --- | --- |
| Need Response | A 接锅型 Accept | R 反抗型 Resist | 面对需求、任务、临时安排时，是倾向接住还是回推 |
| Mental State | S 稳定型 Stable | C 崩溃型 Collapse | 面对压力、批评、变动时，是稳定处理还是明显内耗 |
| Task Style | P 主动型 Proactive | D 极限型 Deadline | 处理任务时，是提前推进还是截止前爆发 |
| Interaction Pattern | U 向上管理型 Upward | H 隐身型 Hidden | 职场互动中，是主动同步包装还是降低存在感 |

---

# 答题方式

每道题是一个职场情境，提供 A/B/C/D 四个不同反应。

注意：

* A/B/C/D 不是固定分值，也不固定代表某个维度方向。
* 每个选项背后都有隐藏计分方向，例如 `A+`、`R+`。
* 每道题只测一个维度，但四个选项会覆盖该维度的两个方向。
* 同方向通常设置一个强倾向选项（5 分）和一个弱倾向选项（4 分）。

这种设计比统一量表更有趣，但仍保持计分逻辑可控。

---

# 计分规则

用户选择某个选项后，将该选项的分值加到对应方向。

示例：

* 某选项计分方向为 `R+`
* 分值为 5
* 用户选择该选项
* 则 `R` 得 5 分

最终分别比较每组维度的得分：

* `A` vs `R`
* `S` vs `C`
* `P` vs `D`
* `U` vs `H`

每组得分更高的一方，就是最终人格代码中的对应字母。

---

# 平分处理规则

如果某一组维度出现平分，按以下规则处理：

1. **仅用该维度下最后两道题**：只根据这两题的用户选项，按本题库「计分方向 / 分值」重新累加该组两侧总分（不是整场 16 题重算）；分数高者胜。
2. 如果仍然平分，选择更符合现代职场默认压力状态的一方：

| 平分维度 | 默认判定 |
| --- | --- |
| A / R | A |
| S / C | C |
| P / D | D |
| U / H | H |

**各维度题目编号（用于第 1 步「最后两题」）**（与根目录 `SKILL.md` 保持一致）

- Need Response：`Q1, Q5, Q9, Q13` → 最后两题为 **Q9、Q13**
- Mental State：`Q2, Q6, Q10, Q14` → **Q10、Q14**
- Task Style：`Q3, Q7, Q11, Q15` → **Q11、Q15**
- Interaction Pattern：`Q4, Q8, Q12, Q16` → **Q12、Q16**

# 题库总览

本题库共 16 题：

* Need Response：4 题
* Mental State：4 题

* Interaction Pattern：4 题

题目采用四维交错顺序，减少连续回答同类题造成的疲劳。

---

# 题目设计原则

1. 每道题是一个真实职场场景，不使用抽象人格描述。
2. 选项必须像真实人会说的话、会做的事，而不是“非常符合/不符合”。
3. 加入老板临时加需求、公开质疑、跨部门卡点、职责甩锅等员工高频冲突。
4. 每道题只测一个维度，避免一个选项同时混入多个维度。
5. 每个维度 4 题，每个方向都通过多种场景重复验证。
6. 文案可以有梗，但不能为了搞笑破坏计分方向。

# 分数解释

每个维度有 4 道题。由于每题只给某一侧加分，每个方向的得分上限取决于用户在该维度下选择该方向的次数和强度。

建议解释：

| 单侧得分差 | 解释 |
| --- | --- |
| 0-3 分 | 倾向较弱，用户可能处在混合状态 |
| 4-8 分 | 倾向明显，结果较可信 |
| 9 分及以上 | 倾向强烈，结果非常典型 |
